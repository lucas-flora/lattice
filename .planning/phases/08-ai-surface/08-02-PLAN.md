# Plan 08-02: Supabase RAG — Vector Table, Embeddings, and Retrieval

## Objective

Set up the Supabase pgvector infrastructure for Retrieval-Augmented Generation. Create the vector table, embedding utilities, RAG retrieval client, and a seed script that populates the table with CA reference material and preset descriptions. This enables the AI to cite relevant documentation when answering questions about rule behavior.

## must_haves

- SQL migration creates `ca_documents` table with `id (uuid)`, `title (text)`, `content (text)`, `embedding (vector(1536))`, `category (text)`, `source (text)`, `created_at (timestamptz)`
- Migration enables the pgvector extension and creates a cosine similarity function
- Embeddings module calls OpenAI `text-embedding-3-small` to generate 1536-dim vectors
- RAG client queries Supabase for top-3 documents by cosine similarity to the user's query embedding
- RAG client gracefully returns empty array if Supabase is unreachable or errors
- Seed script contains CA reference material: GoL patterns (glider, oscillator, still life), reaction-diffusion theory, elementary CA rules, Langton's Ant behavior, Brian's Brain dynamics, Navier-Stokes fluid basics
- Seed script also extracts ai_context from built-in preset YAML files and seeds those as documents

## Tasks

<task id="1">
Create `supabase/migrations/001_ca_documents.sql`:
- Enable pgvector extension: `CREATE EXTENSION IF NOT EXISTS vector;`
- Create `ca_documents` table with columns: `id uuid DEFAULT gen_random_uuid() PRIMARY KEY`, `title text NOT NULL`, `content text NOT NULL`, `embedding vector(1536)`, `category text NOT NULL`, `source text NOT NULL`, `created_at timestamptz DEFAULT now()`
- Create an index on the embedding column for fast cosine similarity: `CREATE INDEX ON ca_documents USING ivfflat (embedding vector_cosine_ops) WITH (lists = 10);`
- Create a Postgres function `match_documents(query_embedding vector(1536), match_threshold float, match_count int)` that returns rows ordered by cosine similarity (1 - cosine distance) above the threshold, limited to match_count
</task>

<task id="2">
Create `src/ai/embeddings.ts`:
- Export `generateEmbedding(text: string): Promise<number[]>` that calls OpenAI Embeddings API
  - Uses `text-embedding-3-small` model (1536 dimensions)
  - Calls `https://api.openai.com/v1/embeddings` via native fetch
  - Reads OPENAI_API_KEY from process.env
  - Returns the embedding array from the response
  - On error, throws with descriptive message
- Export `generateEmbeddings(texts: string[]): Promise<number[][]>` for batch embedding (sends all texts in one API call)
</task>

<task id="3">
Create `src/ai/ragClient.ts`:
- Export `retrieveRelevantDocuments(query: string, options?: { topK?: number, threshold?: number }): Promise<RagDocument[]>`
  - Generates embedding for the query using `generateEmbedding()`
  - Calls the Supabase `match_documents` RPC function via REST API (not the Supabase JS client — use native fetch to avoid adding a dependency)
  - Uses `NEXT_PUBLIC_SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` from process.env (server-side only)
  - Returns top-K documents (default 3) with title, content, category, and similarity score
  - On any error (network, Supabase down, missing env vars), returns empty array with a console.warn — never throws
- Export `RagDocument` type: `{ title: string, content: string, category: string, source: string, similarity: number }`
- Export `formatRagContext(documents: RagDocument[]): string` that formats documents into a string suitable for injection into the system prompt, with citations: "According to [title] ([source]): ..."
</task>

<task id="4">
Create `scripts/seed-rag.ts` as a Node.js script (run via `npx tsx scripts/seed-rag.ts`):
- Define CA reference documents as an array of { title, content, category, source } objects:
  - "Conway's Game of Life Patterns" — describes gliders, oscillators, still lifes, spaceships, and gardens of eden (category: "patterns", source: "ca-reference")
  - "Elementary Cellular Automata" — describes 1D rules, Rule 110 Turing completeness, Wolfram classes (category: "theory", source: "ca-reference")
  - "Reaction-Diffusion Systems" — describes Gray-Scott model, Turing patterns, parameter effects on spots/stripes/waves (category: "theory", source: "ca-reference")
  - "Langton's Ant Behavior" — describes emergent highway behavior, initial chaos, symmetry (category: "patterns", source: "ca-reference")
  - "Brian's Brain Dynamics" — describes three-state automaton, spontaneous pattern formation, glider analogs (category: "patterns", source: "ca-reference")
  - "Navier-Stokes Fluid Simulation" — describes simplified NS equations, viscosity, diffusion, pressure-velocity coupling (category: "theory", source: "ca-reference")
  - "Lattice Command Reference" — describes available commands by category (sim, preset, edit, view, ui) with parameter descriptions (category: "commands", source: "lattice-docs")
- Read the 6 built-in preset YAML files, extract any `ai_context` field from their meta section, and create documents for each (category: "preset", source: "builtin-preset")
- For each document: generate embedding via `generateEmbedding()`, then INSERT into `ca_documents` via Supabase REST API
- Log progress: "Seeded N documents" at the end
- Idempotent: DELETE existing documents by source before inserting (so re-running is safe)
</task>

<task id="5">
Run the Supabase migration and seed script:
- Apply migration via Supabase MCP tools (execute_sql) or via REST API
- Run seed script: `npx tsx scripts/seed-rag.ts`
- Verify documents were inserted by querying the count
</task>
