# Plan 08-03: Terminal Integration — AI Chat, Command Execution, and Typo Correction

## Objective

Wire the AI into the terminal by replacing the "AI not connected" placeholder with a real streaming AI service. Build the client-side AI service that handles streaming SSE responses, parses command actions from AI output, and executes them via CommandRegistry. Add typo detection that routes misspelled commands to AI for correction. This is the plan that makes the AI actually work in the terminal.

## must_haves

- AiService client calls `/api/ai/chat` with user message + context, processes SSE stream, and returns text + optional commands
- AiService builds context via ContextBuilder before each call
- AiService parses AI response for JSON command blocks and executes them via commandRegistry.execute()
- Commands executed by AI appear in terminal log with `[AI]` prefix
- TypoDetector uses Levenshtein distance to detect near-miss commands (short input, 1-3 words, matches command category pattern but doesn't parse as valid command)
- TypoDetector routes suspected typos to AI with a hint, not directly corrected client-side
- useTerminal.executeInput replaces "AI not connected" with real AI call
- aiStore is wired: isLoading is true while streaming, chatHistory accumulates messages
- Loading indicator "thinking..." appears in terminal during AI response

## Tasks

<task id="1">
Create `src/ai/typoDetector.ts`:
- Export `detectPossibleTypo(input: string, commandNames: string[]): { isTypo: boolean, hint: string }`:
  - Split input on whitespace
  - If input has 1-3 words and first word fuzzy-matches a command category (Levenshtein distance <= 2), it's a possible typo
  - If input has 2 words and the dot-joined form fuzzy-matches a command name (Levenshtein distance <= 3), it's a possible typo
  - Return `{ isTypo: true, hint: "Did you mean 'sim play'?" }` with the closest match, or `{ isTypo: false, hint: "" }`
- Export `levenshtein(a: string, b: string): number` utility function for distance calculation
- Command categories derived from command names (unique set of category prefixes)
</task>

<task id="2">
Create `src/ai/aiService.ts`:
- Import types from `./types`, `buildAiContext` from `./contextBuilder`, `commandRegistry` from `@/commands/CommandRegistry`, `useSimStore` from `@/store/simStore`, `aiStoreActions` from `@/store/aiStore`, `detectPossibleTypo` from `./typoDetector`
- Export class `AiService`:
  - `private recentActions: string[]` — rolling buffer of last 10 terminal actions
  - `addRecentAction(action: string)` — push to recentActions, trim to 10
  - `async sendMessage(message: string, options?: { typoHint?: string }): Promise<{ text: string, commands: Array<{ name: string, params: Record<string, unknown> }> }>`
    - Build context via `buildAiContext(useSimStore.getState(), commandRegistry.list(), this.recentActions)`
    - Construct AiChatRequest with message, context, and conversation history from aiStore
    - If `options.typoHint` is provided, prepend it to the message as context for the AI
    - Set aiStore loading to true
    - Call `fetch('/api/ai/chat', { method: 'POST', body: JSON.stringify(request) })`
    - Read the SSE stream, accumulating text deltas
    - On each delta, call `onDelta` callback if provided (for streaming display)
    - After stream completes, parse accumulated text for JSON command blocks
    - Execute any commands found via `commandRegistry.execute()` and collect results
    - Update aiStore: add user message and assistant message to chatHistory, set loading to false
    - Return `{ text: fullResponse, commands: executedCommands }`
  - `async handleTerminalInput(input: string, addLogEntry: (type: string, message: string) => void): Promise<void>`
    - Main entry point called from useTerminal
    - Check typo detection first: `detectPossibleTypo(input, commandRegistry.list().map(c => c.name))`
    - If typo detected: call sendMessage with typoHint, log the AI response and any command executions
    - If not typo: call sendMessage with plain message
    - Stream AI text to terminal as 'ai' type log entries
    - For each command the AI executes, log as `[AI] > {command}` in 'command' type, then log the result
    - On error: log error message and set loading to false
- Export `aiService` singleton instance
</task>
