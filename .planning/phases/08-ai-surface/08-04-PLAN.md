# Plan 08-04: Test Pyramid — Unit, Integration, and Scenario Tests

## Objective

Build comprehensive tests for all Phase 8 AI Surface functionality across all three tiers of the test pyramid. Unit tests verify individual modules in isolation with mocked dependencies. Integration tests verify the full AI pipeline with mocked OpenAI API. Scenario tests verify end-to-end user workflows including command execution via AI and typo correction.

## must_haves

- Unit tests for ContextBuilder: verifies metadata-only output, no grid buffers, correct field mapping
- Unit tests for typoDetector: verifies Levenshtein distance calculation, typo detection accuracy, edge cases
- Unit tests for personality: verifies config structure, system prompt template interpolation
- Unit tests for aiService: verifies streaming, command parsing, error handling (with mocked fetch)
- Unit tests for ragClient: verifies document retrieval, graceful degradation, format function
- Integration tests: full pipeline from user input through context building, API call (mocked), response parsing, command execution
- Scenario tests: natural language message gets AI response, command execution via AI, typo correction workflow, RAG citation in response, no grid state in API payload

## Tasks

<task id="1">
Create `src/ai/__tests__/contextBuilder.test.ts`:
- TestContextBuilder_BuildsMetadataFromSimState: given a simStore state snapshot and command list, builds correct AiContext with all fields populated
- TestContextBuilder_NeverIncludesGridBuffers: verify the output object has NO Float32Array, NO ArrayBuffer, NO property named 'buffer' or 'grid' — only scalar values and string arrays
- TestContextBuilder_MapsPresetNameCorrectly: activePreset maps to presetName in context
- TestContextBuilder_IncludesAvailableCommands: command catalog is mapped to simplified {name, description} list
- TestContextBuilder_DefaultsRecentActionsToEmpty: when no recentActions provided, defaults to []
- TestContextBuilder_TrimsRecentActionsToTen: when >10 recent actions provided, only last 10 are included

Create `src/ai/__tests__/personality.test.ts`:
- TestPersonality_ExportsConfig: PERSONALITY object has name, systemPromptTemplate, tone, maxResponseTokens
- TestPersonality_BuildSystemPrompt_IncludesContext: calling buildSystemPrompt with context includes preset name, generation, etc.
- TestPersonality_BuildSystemPrompt_IncludesRagDocs: when RAG documents are provided, they appear in the prompt
- TestPersonality_BuildSystemPrompt_WorksWithoutRag: when no RAG documents, prompt is valid without them
- TestPersonality_ConfigValues: name is "Lattice AI", tone is "concise-technical"
</task>

<task id="2">
Create `src/ai/__tests__/typoDetector.test.ts`:
- TestLevenshtein_ExactMatch: levenshtein("abc", "abc") === 0
- TestLevenshtein_SingleInsertion: levenshtein("abc", "abcd") === 1
- TestLevenshtein_SingleDeletion: levenshtein("abcd", "abc") === 1
- TestLevenshtein_SingleSubstitution: levenshtein("abc", "axc") === 1
- TestLevenshtein_CompletelyDifferent: levenshtein("abc", "xyz") === 3
- TestLevenshtein_EmptyStrings: handles empty strings correctly
- TestTypoDetector_DetectsSimPlayTypo: "sim plya" is detected as typo with hint "sim play"
- TestTypoDetector_DetectsPresetLoadTypo: "prset load" is detected as typo
- TestTypoDetector_IgnoresNaturalLanguage: "tell me about conway" is NOT detected as typo
- TestTypoDetector_IgnoresValidCommand: "sim play" is NOT detected as typo (it would parse as command)
- TestTypoDetector_HandlesLongInput: "please load the gray scott preset for me" is NOT a typo (>3 words)
- TestTypoDetector_HandlesSingleWordTypo: "smi" is detected as typo for "sim" category
- TestTypoDetector_ReturnsClosestMatch: when multiple commands are close, returns the closest
</task>

<task id="3">
Create `src/ai/__tests__/aiService.test.ts`:
- Mock global fetch to return SSE-formatted responses
- TestAiService_SendMessage_CallsApiWithContext: verify fetch is called with correct URL and body including context metadata
- TestAiService_SendMessage_StreamsResponse: verify accumulated text from SSE stream
- TestAiService_SendMessage_ParsesCommandFromResponse: when AI returns JSON command block, it's extracted and returned
- TestAiService_SendMessage_HandlesNetworkError: on fetch failure, returns error gracefully, sets loading to false
- TestAiService_HandleTerminalInput_RoutesTypoToAi: when typo detected, message includes typo hint
- TestAiService_HandleTerminalInput_LogsCommandExecution: when AI executes a command, it appears in terminal as [AI] > command
- TestAiService_RecentActions_RollingBuffer: addRecentAction keeps only last 10 items
- TestAiService_NoGridStateInRequest: inspect the fetch call body and verify no Float32Array or grid buffer data

Create `src/ai/__tests__/ragClient.test.ts`:
- Mock global fetch for Supabase calls
- TestRagClient_RetrievesDocuments: given a query, returns top-3 documents from mock Supabase response
- TestRagClient_GracefulDegradation_NetworkError: on fetch failure, returns empty array (not throw)
- TestRagClient_GracefulDegradation_MissingEnvVars: when Supabase URL missing, returns empty array
- TestRagClient_FormatRagContext_FormatsWithCitations: formatRagContext produces "According to [title] ([source]): ..." format
- TestRagClient_FormatRagContext_EmptyArray: returns empty string for no documents
</task>

<task id="4">

- Full pipeline tests with CommandRegistry, SimulationController, and mocked OpenAI API
- TestAiSurface_ContextBuilderWithRealStores: load a preset via registry, build context, verify it includes correct preset name and generation
- TestAiSurface_CommandExecutionViaAi: simulate AI returning a command action (preset.load gray-scott), verify the command was executed and simulation state changed
- TestAiSurface_TypoDetectionWithRealCommands: register real commands, test typo detection against actual command catalog
- TestAiSurface_NoGridStateInApiPayload: build context from real simulation state, verify the payload contains ONLY scalar metadata
- TestAiSurface_PersonalityPromptWithContext: build full system prompt with real context, verify it contains preset name and available commands

</task>

<task id="5">
Create `test/scenarios/ai-surface-workflow.test.ts`:
- End-to-end workflow tests simulating full user interactions
- TestAiWorkflow_NaturalLanguageGetAiResponse: user types "what is this simulation?", AI responds with context-aware answer (mock streaming response includes preset name)
- TestAiWorkflow_CommandExecutionViaAi: user types "load gray-scott", AI returns command action, command is executed, simulation state reflects Gray-Scott preset
- TestAiWorkflow_TypoCorrection: user types "sim plya", typo is detected, AI corrects to "sim play", command executes, AI explains the correction
