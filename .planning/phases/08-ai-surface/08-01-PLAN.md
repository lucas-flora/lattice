# Plan 08-01: AI Core — ContextBuilder, Personality, and API Route

## Objective

Build the foundational AI infrastructure: the ContextBuilder that assembles app state metadata for AI calls (never raw grid data), a centralized personality configuration, type definitions, and the Next.js API route that streams GPT-4o responses via SSE. This plan establishes the server-side AI pipeline that all other plans depend on.

## must_haves

- ContextBuilder reads from simStore state + commandRegistry catalog to build a metadata-only context payload
- ContextBuilder NEVER includes raw grid buffer data (Float32Array) — only scalar metadata: preset name, generation, grid dimensions, live cell count, running state, speed, recent actions, available commands
- Personality config exports system prompt template, tone settings, and behavior tuning levers
- API route at `/api/ai/chat` accepts POST with user message + context metadata, streams response via SSE
- API route uses OpenAI Chat Completions API with `stream: true`
- API route reads OPENAI_API_KEY from server-side environment (never exposed to client)
- Type definitions shared between client and server for AI request/response shapes

## Tasks

<task id="1">
Create `src/ai/types.ts` with shared type definitions:
- `AiContext`: metadata payload sent to API (presetName, generation, gridWidth, gridHeight, liveCellCount, isRunning, speed, recentActions: string[], availableCommands: {name: string, description: string}[])
- `AiChatRequest`: { message: string, context: AiContext, conversationHistory?: {role: string, content: string}[] }
- `AiChatResponse`: { type: 'text' | 'command', content: string, command?: { name: string, params: Record<string, unknown> } }
- `AiStreamChunk`: { type: 'delta' | 'done' | 'error' | 'command', content?: string, command?: { name: string, params: Record<string, unknown> } }
- Export all types for use by both client and server
</task>

<task id="2">
Create `src/ai/personality.ts` with centralized personality configuration:
- Export a `PERSONALITY` const object with:
  - `name`: "Lattice AI"
  - `systemPromptTemplate`: A multi-line template string for the GPT-4o system prompt. Include placeholders for context injection (preset name, generation, etc.)
  - The system prompt should establish the AI as a knowledgeable CA assistant that can execute commands, explain simulation behavior, and cite CA reference material
  - Include the instruction that the AI should respond concisely (under 3 sentences for simple queries) and never interrupt the user
  - Include the instruction to return JSON command blocks when the user wants to execute a command, using the format `{"action":"command","name":"<cmd>","params":{...}}`
  - `tone`: "concise-technical" — helpful but not chatty
  - `maxResponseTokens`: 500
  - `includeConversationHistory`: true
  - `maxConversationTurns`: 10
- Export a `buildSystemPrompt(context: AiContext, ragDocuments?: string[])` function that fills the template with actual context values and optionally appends RAG document content
</task>

<task id="3">
Create `src/ai/contextBuilder.ts`:
- Export `buildAiContext(state: SimState, commands: CommandCatalogEntry[], recentActions?: string[]): AiContext`
  - Takes the simStore state snapshot, the command catalog, and optional recent actions
  - Returns an AiContext object with ONLY metadata — no grid buffers, no Float32Arrays
  - Maps simStore fields: generation, activePreset (as presetName), gridWidth, gridHeight, liveCellCount, isRunning, speed
  - Maps commands to simplified list: [{name, description}]
  - Includes recentActions (defaults to empty array)
- Import types from `./types`
- Import `SimState` from `@/store/simStore` and `CommandCatalogEntry` from `@/commands/types` for type references only
- This is a pure function — no side effects, no store subscriptions
</task>

<task id="4">
Create `src/app/api/ai/chat/route.ts` as a Next.js API route:
- Export async `POST(request: Request)` handler
- Parse request body as `AiChatRequest`
- Read `OPENAI_API_KEY` from `process.env` — return 500 if missing
- Import `buildSystemPrompt` from `@/ai/personality` and construct the system prompt with the provided context
- Build the OpenAI messages array: [system prompt, ...conversation history (last N turns), user message]
- Call OpenAI Chat Completions API (https://api.openai.com/v1/chat/completions) with:
  - model: "gpt-4o"
  - stream: true
  - max_tokens: personality.maxResponseTokens
  - temperature: 0.7
- Use native `fetch` to call OpenAI (no SDK dependency) — parse the SSE stream
- Return a `Response` with `ReadableStream` that forwards chunks as SSE (`data: {json}\n\n` format)
- Each chunk is an `AiStreamChunk` — parse OpenAI delta content and forward
- After the full response is assembled, scan for JSON command blocks in the response. If found, emit a final `command` chunk with the parsed command
- On error, emit an `error` chunk and close the stream
- Add appropriate headers: `Content-Type: text/event-stream`, `Cache-Control: no-cache`, `Connection: keep-alive`
</task>
