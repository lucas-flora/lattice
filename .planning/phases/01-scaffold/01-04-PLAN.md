  <name>Task 1: Create Three.js dispose utility with tests</name>
  <files>src/lib/three-dispose.ts, src/lib/__tests__/three-dispose.test.ts</files>
  <action>
  1. Create `src/lib/three-dispose.ts`:

     `disposeObject(obj: THREE.Object3D): void`
     - Recursively traverse the object's children
     - For each Mesh: dispose geometry, dispose material(s), dispose textures found in material properties
     - Handle array materials (multi-material meshes)
     - Handle textures: iterate material properties, dispose any THREE.Texture instances
     - For each non-Mesh child that has geometry or material properties: dispose those too

     `disposeMaterial(material: THREE.Material): void`
     - Call material.dispose()
     - Iterate all keys on the material, dispose any THREE.Texture values

     `disposeRenderer(renderer: THREE.WebGLRenderer): void`
     - Call renderer.dispose()
     - Get WebGL context, call WEBGL_lose_context extension if available

  2. Create `src/lib/__tests__/three-dispose.test.ts`:
     - Use vitest with jsdom environment
     - Test disposeObject with a mock scene tree:
       - Create a THREE.Group with child Meshes containing BoxGeometry and MeshBasicMaterial
       - Create spy functions on geometry.dispose, material.dispose
       - Call disposeObject on the group
       - Assert all dispose methods were called
     - Test with array materials:
       - Create a Mesh with an array of materials
       - Verify each material's dispose is called
     - Test with textures:
       - Create a MeshStandardMaterial with a map texture
       - Verify texture.dispose is called
     - Test empty object (no crash)
     - Test nested hierarchy (grandchildren disposed too)

     Name tests: `TestDisposeObject_DisposesGeometry`, `TestDisposeObject_DisposesArrayMaterials`, `TestDisposeObject_DisposesTextures`, `TestDisposeObject_HandlesEmptyObject`, `TestDisposeObject_TraversesDeepHierarchy`

  Note: Three.js in jsdom won't have a real WebGL context. Mock the renderer for unit tests. The real WebGL test (renderer.info.memory.geometries === 0) is a manual browser verification.
  </action>
  <verify>pnpm vitest run --dir src passes all dispose tests</verify>
  <done>disposeObject utility works, handles all resource types, tests pass with mocked Three.js objects</done>

  <action>
  1. Create four Zustand stores with `subscribeWithSelector` middleware:

     `src/store/simStore.ts`:
     ```typescript
     import { create } from 'zustand';
     import { subscribeWithSelector } from 'zustand/middleware';

     export interface SimState {
       generation: number;
       isRunning: boolean;
       activePreset: string | null;
       gridWidth: number;
       gridHeight: number;
     }

     export const useSimStore = create<SimState>()(
       subscribeWithSelector(() => ({
         generation: 0,
         isRunning: false,
         activePreset: null,
         gridWidth: 0,
         gridHeight: 0,
       }))
     );
     ```

     `src/store/viewStore.ts`: Camera position, zoom level, viewport config stubs
     `src/store/uiStore.ts`: Panel visibility, terminal open state stubs
     `src/store/aiStore.ts`: Chat history array, isLoading boolean stubs
