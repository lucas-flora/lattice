# Plan 07-01: Rust Rule Implementations (Gray-Scott + Navier-Stokes)

## Objective

Implement Gray-Scott reaction-diffusion and Navier-Stokes fluid dynamics rules in Rust with wasm-bindgen exports. Each rule function operates on the full grid buffer in a single call per tick (whole-tick API, RULE-04). The Rust implementations match the TypeScript compute body logic exactly, producing identical output for the same inputs.

## must_haves

- Gray-Scott `gray_scott_tick()` function in Rust that processes an entire 2D grid in one call
- Navier-Stokes `navier_stokes_tick()` function in Rust that processes an entire 2D grid in one call
- Both functions accept `&mut [f32]` slices for each property buffer plus grid dimensions and parameters
- Both functions use toroidal wrapping for neighbor lookups
- Laplacian computation matches the TypeScript implementation (scaled by 4.0/neighborCount)
- All functions are `#[wasm_bindgen]` exported for JS consumption
- Rust unit tests verify correct output after N ticks for both rules
- Rust unit test confirms exactly 1 extern boundary crossing per tick (single function call processes all cells)
- `console_error_panic_hook` added for WASM debugging
- Grid utility module for shared neighbor indexing with toroidal wrapping

<task id="3">
Create `crates/lattice-engine/src/gray_scott.rs`:
- Implement `gray_scott_tick(u_buf: &mut [f32], v_buf: &mut [f32], u_out: &mut [f32], v_out: &mut [f32], width: u32, height: u32, du: f64, dv: f64, f: f64, k: f64, dt: f64)`:
  - Reads from u_buf/v_buf (current state), writes to u_out/v_out (next state)
  - For each cell: compute Laplacian of U and V using Moore neighborhood with toroidal wrap
  - Scale Laplacian by 4.0/8.0 (8 Moore neighbors)
  - Apply reaction-diffusion equations: newU = u + dt * (Du * lapU - u*v*v + F*(1-u)), newV = v + dt * (Dv * lapV + u*v*v - (F+k)*v)
  - Clamp output to [0, 1]
  - Exact match to TypeScript compute body in gray-scott.yaml
- Export with `#[wasm_bindgen]` -- wasm-bindgen maps `&mut [f32]` to JS Float32Array
- Unit tests:
  - TestGrayScott_SingleTick_MatchesExpected: verify output for a known input pattern
  - TestGrayScott_MultipleTicks_ProducesNonTrivial: run 10 ticks on a seeded grid, verify output differs from initial state
  - TestGrayScott_ClampOutput: verify values stay in [0, 1]
</task>

<task id="4">
Create `crates/lattice-engine/src/navier_stokes.rs`:
- Implement `navier_stokes_tick(vx: &mut [f32], vy: &mut [f32], density: &mut [f32], pressure: &mut [f32], vx_out: &mut [f32], vy_out: &mut [f32], density_out: &mut [f32], pressure_out: &mut [f32], width: u32, height: u32, viscosity: f64, diffusion: f64, dt: f64)`:
  - Reads from input buffers, writes to output buffers
  - For each cell: compute Laplacians of vx, vy, density; compute pressure gradient and velocity divergence
  - Apply simplified NS equations matching TypeScript compute body in navier-stokes.yaml
  - Apply damping (0.999) and clamping (velocities to [-10, 10], density to [0, 10], pressure to [-10, 10])
- Export with `#[wasm_bindgen]`
- Unit tests:
  - TestNavierStokes_SingleTick_MatchesExpected: verify output for a known input
  - TestNavierStokes_MultipleTicks_ProducesNonTrivial: run 10 ticks, verify output differs
  - TestNavierStokes_Clamping: verify output values stay within bounds
</task>
