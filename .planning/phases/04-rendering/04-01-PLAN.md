  Create `src/renderer/VisualMapper.ts`:

  The VisualMapper reads `visual_mappings` from a PresetConfig and provides fast lookup functions for each visual channel. It is a pure data transformation layer — no Three.js dependencies.

  ```typescript
  import * as THREE from 'three';
  import type { PresetConfig } from '@/engine/preset/types';

  export class VisualMapper {
    private colorMaps: Map<string, Map<string, THREE.Color>>;
    private sizeMaps: Map<string, Map<string, number>>;
    private orientationMaps: Map<string, Map<string, number>>;
    private defaultColor: THREE.Color;
    private defaultSize: number;

    constructor(preset: PresetConfig) { ... }

    // Get color for a property value (returns default if no mapping)
    getColor(propertyName: string, value: number): THREE.Color { ... }

    // Get size scale factor for a property value
    getSize(propertyName: string, value: number): number { ... }

    // Get orientation in radians for a property value
    getOrientation(propertyName: string, value: number): number { ... }

    // Check if a property has a color mapping
    hasColorMapping(propertyName: string): boolean { ... }

    // Get the first property that has a color mapping (for default rendering)
    getPrimaryColorProperty(): string | null { ... }
  }
  ```

  Key implementation details:
  1. Parse `preset.visual_mappings` array in constructor
  2. For color mappings: convert hex strings to THREE.Color objects and store in a Map keyed by string representation of property value
  3. For size mappings: store numeric scale factors
  4. For orientation mappings: store radian values
  5. Default color: black (#000000). Default size: 1.0.
  6. Value lookup uses String(Math.round(value)) for boolean/int properties (matching YAML keys like "0", "1")
  7. If no visual_mappings in preset, provide a default mapping: first bool property -> color green/black

  Create `src/renderer/LatticeRenderer.ts`:

  The LatticeRenderer is the unified Three.js rendering engine for all grid dimensionalities. It uses InstancedMesh for efficient GPU-instanced rendering and reads typed arrays directly from the Grid.

  ```typescript
  import * as THREE from 'three';
  import type { Grid } from '@/engine/grid/Grid';
  import type { PresetConfig } from '@/engine/preset/types';
  import { VisualMapper } from './VisualMapper';
  import { disposeObject, disposeRenderer } from '@/lib/three-dispose';

  export class LatticeRenderer {
    readonly scene: THREE.Scene;
    readonly camera: THREE.OrthographicCamera;
    readonly renderer: THREE.WebGLRenderer;
    private instancedMesh: THREE.InstancedMesh | null = null;
    private visualMapper: VisualMapper;
    private grid: Grid | null = null;
    private preset: PresetConfig | null = null;
    private cellSize: number = 1;
    private tempMatrix: THREE.Matrix4;
    private tempColor: THREE.Color;

    // 1D spacetime state
    private historyBuffers: Float32Array[] = [];
    private maxHistory: number = 128;
    private renderMode: '2d' | '1d-spacetime' = '2d';

    constructor(config: RendererConfig) { ... }

    // Initialize or reinitialize for a new simulation
    setSimulation(grid: Grid, preset: PresetConfig): void { ... }

    // Update instance matrices and colors from current grid state
    update(): void { ... }

    // Render one frame
    render(): void { ... }

    // Resize the renderer and camera
    resize(width: number, height: number): void { ... }

    // Dispose all GPU resources
    dispose(): void { ... }

    // Get renderer info for memory leak verification
    getMemoryInfo(): { geometries: number; textures: number } { ... }
  }
  ```

  Key implementation details:

  1. **Constructor**: Create WebGLRenderer (with antialias option), Scene (background black), OrthographicCamera. Reuse a single `tempMatrix` (Matrix4) and `tempColor` (Color) for per-instance updates — no per-frame allocation.

  2. **setSimulation()**:
     - Dispose existing instancedMesh if any
     - Determine render mode: '1d-spacetime' for 1D grids, '2d' for 2D grids
     - Create PlaneGeometry(1, 1) for cell quads
     - Create MeshBasicMaterial({ vertexColors: true }) for instanced colors -- NOTE: In Three.js r183, for InstancedMesh with setColorAt, you may need to set `material.vertexColors` or just rely on instanceColor attribute
     - For 2D: instanceCount = grid.cellCount
     - For 1D spacetime: instanceCount = grid.config.width * maxHistory
     - Create InstancedMesh(geometry, material, instanceCount)
     - Initialize all instance positions using grid.indexToCoord()
     - Create VisualMapper from preset
     - Set camera frustum to frame the grid (zoom to fit on load)

  3. **update()**:
     - If no grid, return
     - Get primary color property from VisualMapper
     - Read buffer directly: `const buffer = this.grid.getCurrentBuffer(propertyName)` — ZERO COPY (RNDR-12)
     - For 2D mode: iterate cells, lookup color from VisualMapper, call setColorAt()
     - For 1D spacetime: snapshot current generation buffer, push to history, rebuild all instances from history
     - Set instanceMatrix.needsUpdate = true, instanceColor.needsUpdate = true
     - For size mappings: update scale in instance matrix via tempMatrix

  4. **dispose()**:
     - Call disposeObject(this.scene) to clean all scene children
     - Call disposeRenderer(this.renderer)
     - Null out references

  5. **getMemoryInfo()**: Return `{ geometries: this.renderer.info.memory.geometries, textures: this.renderer.info.memory.textures }`
