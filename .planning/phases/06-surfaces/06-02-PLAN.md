# Plan 06-02: Terminal Component

## Objective

Build the CLI terminal component that slides up from the bottom of the screen. It displays curated app logs, accepts CLI commands with ghost-text autocomplete, parses commands into registry calls, and routes non-command input to the AI assistant placeholder hook.

## must_haves

- Terminal panel toggles via backtick key or Ctrl+` or ui.toggleTerminal command
- Output log shows timestamped, color-coded entries (command=cyan, info=gray, error=red, AI=purple)
- Command input with prompt `>`, blinking cursor, command history (up/down arrows)
- Ghost-text autocomplete with Tab to accept
- Command parsing: `sim play` -> `sim.play`, `preset load conways-gol` -> `preset.load { name: "conways-gol" }`
- Non-command text routes to AI placeholder returning "AI not connected" message
- Terminal is generic shell infrastructure; AI is just a consumer

## Tasks

<task id="1">
Create `src/components/terminal/commandParser.ts`:
- `parseCommand(input: string)`: parses `category action args` format into `{ commandName: string, params: Record<string, unknown> }` or null if not a command
- Maps: `sim play` -> `sim.play {}`, `sim speed 30` -> `sim.speed { fps: 30 }`, `preset load conways-gol` -> `preset.load { name: "conways-gol" }`, `edit draw 5 5` -> `edit.draw { x: 5, y: 5 }`, `edit brush-size 3` -> `edit.brushSize { size: 3 }`
- Hyphenated actions convert to camelCase: `step-back` -> `stepBack`, `brush-size` -> `brushSize`
- `getAutocompleteSuggestions(input: string, registry: CommandRegistry)`: returns matching command suggestions for ghost-text
- `isCommand(input: string, registry: CommandRegistry)`: checks if input starts with a valid command prefix
</task>

<task id="2">
Create `src/components/terminal/useTerminal.ts` -- custom React hook:
- Manages terminal state: output log entries, command history, current input, ghost-text suggestion
- `addLogEntry(type, message)`: adds timestamped entry to output
- `executeInput(input)`: parses command, executes via registry, adds result to log. Non-commands route to AI placeholder.
- `navigateHistory(direction)`: up/down arrow through command history
- `getGhostText(input)`: returns autocomplete suggestion for current input
- Subscribes to EventBus to capture curated events as log entries (command executions, state changes, errors)
- Max scrollback: 500 entries
</task>

<task id="3">
Create `src/components/terminal/TerminalOutput.tsx`:
- Scrollable output area showing log entries
- Each entry: timestamp (HH:MM:SS, gray), message text colored by type
- Colors: command=cyan (#22d3ee), info=gray (#a1a1aa), error=red (#ef4444), AI=purple (#a78bfa)
- Auto-scrolls to bottom on new entries
- Monospace font throughout
</task>

<task id="4">
Create `src/components/terminal/TerminalInput.tsx`:
- Input line with `>` prompt character
- Ghost-text overlay: dimmed text showing autocomplete suggestion after cursor
- Tab key accepts ghost-text suggestion
- Enter key submits command
- Up/down arrows navigate command history
- Escape clears input
- Prevent default on backtick to avoid toggling terminal while typing
</task>

<task id="5">
Create `src/components/terminal/Terminal.tsx`:
- Main terminal panel component
- Slides up from bottom of screen, default height 30vh
- Contains TerminalOutput and TerminalInput
- Dark background (zinc-900), semi-transparent top border
- Resize handle on top edge for manual height adjustment
- Keyboard shortcut listener: backtick or Ctrl+` toggles terminal via ui.toggleTerminal command
</task>
