# Plan 06-04: AppShell Layout, Cell Drawing, and Viewport Integration

## Objective

Rewrite the main page to integrate all surfaces into a unified AppShell. Refactor SimulationViewport to use the shared SimulationController instead of a local Simulation. Add cell drawing/erasing on the grid. Create integration and scenario tests for the full surface layer.

## must_haves

- AppShell wraps all components: SimulationViewport, ControlBar, HUD, Terminal, ParamPanel, PresetSelector
- SimulationViewport uses the shared SimulationController (not a local Simulation)
- Cell drawing: left-click draws, right-click erases, configurable brush size
- Drawing auto-pauses simulation and records as undoable command
- Mouse cursor changes to crosshair in draw mode
- Grid coordinates calculated from screen position via camera
- On app mount: create EventBus, SimulationController, register all commands, wire stores, load default preset
- Client-side preset loading (no fs.readFileSync in browser)
- Integration and scenario tests

<task id="2">
Create `src/components/AppShell.tsx`:
- Top-level layout component that initializes the command infrastructure
- On mount: create EventBus, SimulationController, register all commands, wire stores
- Load default preset (conways-gol) on startup
- Provide controller and registry to children via React context or module-level singleton
- Render: SimulationViewport (full-screen background), HUD (top-left overlay), PresetSelector (top-right overlay), ControlBar (bottom-center overlay), Terminal (bottom slide-up), ParamPanel (right slide-in)

</task>

<task id="3">
Refactor `src/components/viewport/SimulationViewport.tsx`:
- Remove local Simulation creation -- use SimulationController from context/singleton
- Subscribe to EventBus events for simulation state changes
- On `sim:presetLoaded`: reinitialize renderer with new grid
- On `sim:tick`: update renderer (renderer.update() + renderer.render())
- Add cell drawing support:
  - Convert screen coordinates to grid coordinates via camera unprojection
  - Left-click: begin draw command, set cells alive
  - Right-click: begin erase command, set cells dead
  - Mouse drag: continue drawing/erasing in stroke
  - Mouse up: commit command to CommandHistory
  - Apply brush size from uiStore.brushSize
  - Auto-pause on first draw if running
  - Change cursor to crosshair during draw
- Continue to handle pan/zoom (middle-click or shift+click for pan, wheel for zoom)
</task>
