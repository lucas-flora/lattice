# Plan 05-02: SimulationController and Command Definitions

## Objective

Build the SimulationController that wraps the Simulation class with play/pause lifecycle and event emission, then register all commands in the CommandRegistry. This is where `CommandRegistry.execute("sim.play", {})` actually starts the simulation.

## must_haves

- SimulationController wraps Simulation with play/pause/step/reset/loadPreset
- SimulationController emits events via EventBus on every state change
- All commands registered: sim.play, sim.pause, sim.step, sim.reset, preset.load, view.zoom, view.pan, view.fit, edit.undo, edit.redo, ui.toggleTerminal, ui.toggleParamPanel
- CommandRegistry.execute("sim.play", {}) starts the simulation through the controller
- Every registered command has complete metadata (name, description, category, params schema)
- A command invoked via the registry produces identical engine state change as invoking the underlying engine method directly

## Tasks

<task id="1">
Create `src/commands/SimulationController.ts`:
- Class that wraps a Simulation instance and manages the play/pause tick loop
- Constructor takes EventBus instance (for testability -- don't rely on singleton in tests)
- `loadPreset(presetName: string)`: loads a preset via loadBuiltinPreset, creates new Simulation, emits `sim:presetLoaded`
- `play()`: starts tick loop using setInterval (not requestAnimationFrame -- must work in Node.js tests), emits `sim:play`
- `pause()`: stops tick loop, emits `sim:pause`
- `step()`: runs one tick, emits `sim:tick`
- `reset()`: resets simulation to generation 0, emits `sim:reset`
- `isPlaying()`: returns current play state
- `getGeneration()`: returns current generation
- `getSimulation()`: returns the underlying Simulation instance (for direct comparison in tests)
- Tick loop: on each interval, call simulation.tick(), emit `sim:tick` with generation
- Default tick interval: 100ms (configurable)
</task>
