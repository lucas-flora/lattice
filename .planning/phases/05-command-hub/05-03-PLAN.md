# Plan 05-03: Store-Event Wiring and Integration Tests

## Objective

Wire all four Zustand stores to the EventBus so they update reactively when the engine emits events. Add integration and scenario tests that verify the full command-to-store pipeline and event sequence recording.

## must_haves

- simStore updates generation, isRunning, activePreset, gridWidth, gridHeight from engine events
- viewStore updates zoom, cameraX, cameraY from view:change events
- uiStore updates isTerminalOpen, isParamPanelOpen from ui:change events
- aiStore subscription point established (no AI events yet, but wiring function exists)
- wireStores() connects all stores to EventBus in one call
- A subscriber test records event sequence and verifies correct order (Success Criterion #4)
- Integration test: full command -> event -> store update pipeline
