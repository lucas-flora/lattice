# Plan 05-01: EventBus and CommandRegistry Core

## Objective

Build the two foundational infrastructure pieces: a typed EventBus for engine-to-store communication, and the CommandRegistry that serves as the architectural hub for all app actions.

## must_haves

- EventBus emits and receives typed events with correct payloads
- CommandRegistry.register() accepts command definitions with name, description, category, params schema, and execute function
- CommandRegistry.list() returns a typed catalog of all registered commands
- CommandRegistry.execute(name, params) validates params and runs the command's execute function
- CommandRegistry.get(name) returns a single command definition
- All commands are async (return Promise<CommandResult>)
- Invalid params produce typed error results, never throw

## Tasks

<task id="1">
Create `src/commands/types.ts` with TypeScript type definitions:
- `CommandDefinition`: name (string), description (string), category (string), params (ZodSchema), execute (function returning Promise<CommandResult>)
- `CommandResult`: { success: boolean, data?: unknown, error?: string }
- `CommandCatalogEntry`: name, description, category, params schema description (for list() output)
</task>

<task id="2">
Create `src/engine/core/EventBus.ts`:
- Typed event emitter class with `on(event, handler)`, `off(event, handler)`, `emit(event, payload)` methods
- Event type map: `sim:tick` -> `{ generation: number }`, `sim:play` -> `{}`, `sim:pause` -> `{}`, `sim:reset` -> `{}`, `sim:presetLoaded` -> `{ name: string, width: number, height: number }`, `view:change` -> `{ zoom?: number, cameraX?: number, cameraY?: number }`, `ui:change` -> `{ isTerminalOpen?: boolean, isParamPanelOpen?: boolean }`, `edit:undo` -> `{}`, `edit:redo` -> `{}`
- Export singleton `eventBus` instance and the `EventBus` class
- Pure TypeScript, zero UI imports
- Update `src/engine/core/index.ts` to re-export EventBus and event types
</task>

<task id="3">
Create `src/commands/CommandRegistry.ts`:
- Singleton class with private Map<string, CommandDefinition> for registered commands
- `register(definition: CommandDefinition)`: validates definition has required fields, stores in map, throws if duplicate name
- `list()`: returns array of CommandCatalogEntry (name, description, category, params schema)
- `get(name)`: returns CommandDefinition or undefined
- `execute(name, params)`: looks up command, validates params against Zod schema, calls execute function, catches errors and wraps in CommandResult
- `clear()`: for testing -- removes all registered commands
- Export singleton `commandRegistry` instance and CommandRegistry class
</task>

<task id="4">
Create `src/commands/index.ts`:
- Re-export CommandRegistry, commandRegistry singleton, types
- Re-export EventBus and eventBus from engine/core
</task>

<task id="5">
Create `src/engine/core/__tests__/event-bus.test.ts`:
- TestEventBus_EmitsTypedEvents: emit sim:tick with generation, handler receives correct payload
- TestEventBus_MultipleListeners: two handlers on same event both called
- TestEventBus_OffRemovesListener: handler not called after off()
- TestEventBus_DifferentEvents: handler for sim:tick not called when sim:pause emits
- TestEventBus_EmitWithNoListeners: no error when emitting with no listeners
- TestEventBus_TypeSafety: verify event payloads match type map (compile-time check via test structure)
</task>

<task id="6">
Create `src/commands/__tests__/command-registry.test.ts`:

- TestCommandRegistry_ListReturnsTypedCatalog: list() returns name, description, category, params for each command

- TestCommandRegistry_ExecuteValidatesParams: execute with invalid params returns error result, does not throw
- TestCommandRegistry_ExecuteUnknownCommand: execute with unknown name returns error result
- TestCommandRegistry_GetReturnsDefinition: get() returns the registered command
- TestCommandRegistry_GetUnknownReturnsUndefined: get() returns undefined for unknown name
